The ButtonLed system: Requirements

  1. Design and build a ButtonLed software system (BLS) in which a Led is turned on and off each time a Button is pressed (by an human user). The system should run on a a single support, e.g. aConventional PC, a RaspberryPi or Arduino.
  2. After step 1, modify the software system so that the Led could be turned on/off by a remote program.

Basic operational steps (just to start ...)

  1. Open an new Eclispe workspace.
  2. Create the project it.unibo.buttonLed
  3. Create the packages it.unibo.buttonLed, it.unibo.buttonLed.interfaces, it.unibo.buttonLed.impl, it.unibo.buttonLed.docs
  4. Create the project it.unibo.buttonLed.test that depends on the previous one and on JUnit.
  5. Define the domain model, i.e. the models of the entities Led and Button as conventional "Plain Old Java Object" (POJO). The model must include the interfaces ILed and IButton (written in Java on in the selected implementation language).
  6. Define (in the it.unibo.buttonLed.test project) the test plans for the interfaces ILed and IButton.
