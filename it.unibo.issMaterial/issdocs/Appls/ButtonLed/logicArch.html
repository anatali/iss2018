A first (oo) logical architecture

From the analysis of the requirements we know that:
  1. The system is composed of two objects: a Led and a Button.

    Starting form the model of a button as an observable input device, the logic architecture of our software system can be initially defined in the conceptual space of "classic" object oriented software development as shown in the following informal interaction diagram:
  2. The Led is (modelled as) a passive output device, while the Button is (modelled as) as an active observable input device. Thus we exclude the previous interaction. In any case, we need another component to configure the system.
  3. The responsibility of the application logic cannot be given to the devices; thus such a responsibility must be given to a new component (of interface IBLSControl).

Since all the application logic is performed within the thread of the input device, the controller should return the control as soon as possible.
