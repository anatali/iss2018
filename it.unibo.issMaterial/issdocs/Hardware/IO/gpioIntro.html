GPIO

GPIO (generalpurpose I/O pins) enable the software to either read or write voltage levels representing a logical zero or one. In active high logic, a voltage close to VDD (the processor supply voltage) represents a logical one, and a voltage near zero represents a logical zero. In active low logic, these interpretations are reversed.

GPIO as an output

When a GPIO pin is configured to be an output, this enables software to write to a memory-mapped register to set the output voltage to be either high or low. By this mechanism, software can directly control external physical devices. If a GPIO pin produces an output voltage of VDD when given a logical one, then the designer needs to know the current limitations before connecting a device to it. If a device with a resistance of R ohms is connected to it, then the output current will be I=VDD/R (Ohmâ€™s law). It is essential to keep this current within specified tolerances. Going outside these tolerances could cause the device to overheat and fail. A power amplifier may be needed to deliver adequate current. An amplifier may also be needed to change voltage levels.

GPIO as an intput

GPIO pins can also be configured as inputs, in which case software will be able to react to externally provided voltage levels. An input pin may be Schmitt triggered, in which case they have hysteresis. A Schmitt triggered input pin is less vulnerable to noise It may be important to maintain electrical isolation between processor circuits and external devices. A useful strategy is to divide a circuit into electrical domains, possibly with separate power supplies, that have relatively little influence on one another.
