Ingegneria dei sistemi Software: Final Task 2016/17

The environment

A differential drive robot (called from now on robot) is equipped with a sonar put in its front. The robot works in a pyhisical environment like that reported in the following picture:



This scene is taken from a Unity-based system. The two sonars are situated along a straight line along the W-E direction and each sonar is modeled as an active device that emits (in QActor semantics) events of the form: 
    sonar  : sonar(SONARNAME, TARGETNAME, DISTANCE)
At some distance dx, 0<dx<dm (dm prefixed, dx unknown) from the sonar line, there are fixed and mobile obstacles like those shown in the scene. The MobileObstacle moves backwards and forwards along the N-S direction with a speed such as it completes its periodic movement in a prefixed time ot (0<ot<5 seconds).

Functional Requirements: STEP 1

Design and build a (prototype of a) software system that:
  1. starts the robot (already put in A) when the user sends to it a start command by using a remote console;
  2. drives the robot along a straight path from A to B if no onstacle is detected;
  3. avoids the mobile obstacle by waiting (when detected) that the obstacle disappears from the robot's front;
  4. avoids the fixed obstacles by finding some alternative path to reach its goal;
  5. stops the robot if an alarm event occurs.

Functional Requirements: STEP 2 (optional)

After completion of the first step, extend the system built at step1 so that a human user can take the control at any time after robot starting. In this case:
  1. the robot must stop its journey and execute movement commands sent by the user via the console. Obviously the robot should avoid to hit obstacles;
  2. among these commands, the user can tell the robot to return in autonomous way to its starting point along the same path already covered, by avoiding to hit (mobile) obstacles.
