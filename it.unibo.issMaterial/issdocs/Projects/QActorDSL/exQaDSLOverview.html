Actors and plan actions
A QASystem with two Actors running 'in parallel' in the same Context.
One actor executes a Plan that 'calls' another plan that repeats itself and then 'resumes' the previuos one. 		A Qactor is an active (autnomous) entity that runs in parallel with the other actors of the same context.
A Plan is a sequence of Move that can also transfer the control to another plan, either in autonomous way or as a consequence of events. Plan switch is different from a procedure call, since there are no arguments or return values; moreover a plan returns the control to the caller only when the resumeLastPlan key is specified.

Guards
A Qactor executes actions accroding to conditions expressed as guards
Each QActor owns a private WorldTheory used to compute Guards The content of the WorldTheory can be statically intitialized with the Rules specification or dynamically extended via the addRule operation ex05(Guards)
