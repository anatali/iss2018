We report here the basic steps that we have followed to overcome the abstraction gap detected after the analysis phase of a number of distributed proactive/reactive systems :

  1. Project it.unibo.qactor:
    Run time support for QActor including reactive long actions (see QActorHello.java);
  2. Project it.unibo.xtext.qactor:
    1. Specification of a custom language(metamodel) to express QActor concepts (see QActorHello and CtxOfHello) ;
    2. Definition of a code generator (see also QactorRuntimeModule) that generates the behavior code for a Java machine;
    3. Definition of a code generator) that generates the behavior code (see plans.txt and QActorPlanTest.java ) for a custom interpreter (see QActorPlanned.java);
