We report here the basic steps that we have followed to overcome the abstraction gap detected after the analysis phase of a robot-movement problem like that reported in concreteProblems :

  1. Project it.unibo.qactor:
    Run time support for QActor including reactive long actions (see QActorHello.java);
  2. Project it.unibo.xtext.qactor:
    Specification of a custom language(metamodel) to express QActor concepts;
  3. Project it.unibo.xtext.qactor:
    Definition of a code generator) that generates the behavior code for a Java machine;
  4. Project it.unibo.xtext.qactor.robot:
    Definition of a RobotActor.java as a QActor actor that executes also robot movement actions
