The robot (meta)problem

A (differential drive) robot is able to execute a set of move-commands. A move-command can be sent by some element external to the robot, called Mind.

The robot Mind can run on the same computational support of the robot or on a different computational node. In the first case we say the robot is controlled in Embedded-mode, while in second case we say that the ronbot is controlled in Avatar-mode.

A sequence of movement commands emitted by the Mind is called a Plan. During the execution of a Plan, the robot must be able to:
  • react to alarms emitted by other Minds, by other robots, by sensors embedded in the robot or by external devices (e.g. a radar);
  • accept (as soon as possible) a command (called Halt) able to immediately block any movement in the robot, whatever be its current task, including the handling of an alarm.
The handling of an alarm can be a simple (atomic) move (e.g. a stop) or the execution of some alternative plan. When an alternative (alarm handling) plan is ternminated, the robot could return to (resume) the completion of the plan interrupted by the alarm.
