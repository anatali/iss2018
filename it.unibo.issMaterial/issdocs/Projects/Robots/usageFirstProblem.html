A first problem in the (custom) robot field

Design and build an application that allows users to command the movements of a robot by using some remote input device. We are particularly interested in two input devices:

  • A conventional keyboard using the chars: a,s,d,w,q where (with prefixed speed)
    • q means STOP
    • a means LEFT
    • s means BACKWARD
    • w means FORWARD
    • d means RIGHT
  • A GUI interface like the following one:

Analysis

  • Whatever be the user interface, the remote input device can be viewed as a geenrator of messages or events.
  • The mapping from a user command to a robot command can be delegated to an Interpreter.
  • The robot command model should be given by the robot system designer.
