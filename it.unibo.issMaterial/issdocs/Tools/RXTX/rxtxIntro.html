RXTX

In recent weeks, users with 64-bit Windows machines have encountered problems with Java when launching the Programming Window. The solution is to ensure that you have the correct RXTX library files installed in the correct folders. There are two files: rxtxComm.jar and rxtxSerial.dll, which should be installed as follows:

  • C:\Program Files\Java\jre6\bin\rxtxSerial.dll
  • C:\Program Files\Java\jre6\lib\ext\RXTXComm.jar
Many 64-bit Windows machines have both 32-bit and 64-bit browsers and 32-bit and 64-bit Java. On such machines, the 64-bit libraries should be installed as shown above. The correct path for the 32-bit RXTX libraries is as follows:
  • C:\Program Files(x86)\Java\jre6\bin\rxtxSerial.dll
  • C:\Program Files(x86)\Java\jre6\lib\ext\RXTXComm.jar
The installer package on the Machine Science web site attempts to detect whether you have a 32-bit or a 64-bit machine, and install the files as needed. If this fails, the solution is to download and install the files manually. p> The 32-bit RXTX files are here:
  • rxtxSerial.dll
  • RXTXComm.jar
p> The 64-bit RXTX files are here:
  • rxtxSerial.dll
  • RXTXComm.jar

Browswer

The important thing with Windows systems is to make sure that you have the right type of Java for your browser, and the right RXTX files for your version of Java. In other words, if you have a 32-bit browser, you need to install 32-bit Java, and then install the 32-bit RXTX files. If you have a 64-bit browser, you need to install 64-bit Java, and then install the 64-bit RXTX files. Note that many 64-bit Windows machines ship with both 32-bit and 64-bit versions of Internet Explorer.
